解决STRUTS2 错误 There is no Action mapped for namespace / and action name login

By NeroHwang

Feb 18 2014 Updated:Feb 18 2014

Contents

    struts文件名和文件位置要确定是正确的

    首先，确定struts配置的文件名是struts.xml，不要写成任何其他文件名。
    第二，如果你是在tomcat服务器里面部署web应用，那么struts.xml配置文件所在的文件夹位置应该放在webroot--WEB-INF--classes文件夹里面。
    其中，Web应用的基本骨架如下：


    |----WEB-INF
    | |----classes
    | |----lib
    | |----web.xml
    |-index.jsp
    如果你是在eclipse中通过虚拟服务器开启这个web应用，那么要注意，这个struts.xml文件应该在项目的src文件夹内。

    build文件夹里面的文件就是项目编译运行之后生成的文件，全部来自src文件夹。build文件夹中的内容就是你在部署web应用的时候，需放在classes文件夹内的文件。

    其实当我遇到这个问题的时候，也是不属于上述任何一种。因为这些小细节我是一直都很注意的，小细节不注意，会做很多无用功。
    最后我发现自己的问题出在eclipse身上。
    我用的是ubuntu14.04系统，重装过一次以后，放tomcat的文件夹和工作空间那个文件夹都还在，因为使用的是Eclipse kelper jee的版本，只要在系统上配置好Java runtime就可以解压即用的。
    eclipse直接读取那个工作空间目录的全部项目，正常。但是在新增tomcat7服务器的时候发现竟然不给装载！然后就用了eclipse默认提供的j2ee服务器，问题应该就出在这里。
    我不知道它的j2ee服务器路径是在哪里的，所以也就没办法排错。
    后来找到一个方法，将工作空间文件夹内的隐藏文件夹.metadata删除。
    注意！上述操作可能会导致你的eclipse无法正常打开！
    但是由于我的eclipse是直接解压就可以用的，所以删掉.metadata文件夹以后我重新解压一个eclipse拿来用就可以了，同样是选择原来的工作空间目录，正常开启eclipse以后会重新生成.metadata文件夹。
    这个时候就能正常自行选择装载哪个tomcat服务器了，上述struts访问出错的问题也不复存在。
